
## Digit denoting

In a binary digit, we denote the last bit as LSB (least significant bit) and the first bit MSB (most ...)

For signed integers, we preserve the MSB for sign, and flip every bit to present negative integers (one's complement)

In practice we use Two's complement. We present negative integers by ensuring $x + (-x) = 0$ in binary normal addition, ignoring carry out. So we flip every bit and add one to present negative integers.

**Sign Extension (Sext)**: set the new bits with the original MSB. 

**Overflow**: we have overflow if signs of both operands are the same and sign of the sum is different. In hard ware carry into MS bit does not equal carry out when overflow happens.

**Fractions** Floating Point (equivalent of "scientific notation")

32 Bit float point (**FP32**): One bit for sign, 8 Bit for exponent, and 23 Bit for fraction (_mamtissa_). And 

$$
\begin{cases}
    N = (-1)^S \times [1.\text{fraction}]_2 \times 2^{\text{exponent} - 127} &,\quad 1 \leq \text{exponent} \leq 254 \\  
    N = (-1)^S \times [0.\text{fraction}]_2 \times 2^{-126} &, \quad \text{exponent} = 0 \\
    \text{NaN}(\pm \infty) &, \quad \text{exponent} = 255
\end{cases} 
$$

In this case, the number of the numbers can be denoted between $2^k, 2^{k + 1}$ is $2^{23}$, so the precision gets lower when numbers gets larger.
