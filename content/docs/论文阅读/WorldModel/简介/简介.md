
## World Model

World model learns the **Environment dynamics**:

$$
s_{t + 1} = f(s_t, a_t)
$$

i.e. given the current state $s_t$ and the current action $a_t$ on the agent, the model should output the next world state $s_{t + 1}$.

!!! remarks "On the contrary, RL is $s_t$ --> $a_t$"

### World Model in RL

Article: [![arXiv](https://img.shields.io/badge/arXiv-2505.14357-b31b1b.svg)](https://arxiv.org/pdf/1912.01603)

The RL pipeline with world model in Dreamer:

!!! pure ""

    Real world data --> Train world model --> Get imagination rollout --> Train the RL

### World Model in Video Generation

Pipeline:

!!! pure ""

    Current frame + User's action --> Next frame/Future frames

It is always used for AI game engine.

### Robot World Model

Always used for robot actions like capturing, pushing and manipulation --> Driving

Pipeline:

!!! pure ""

    Vision + Robot state + action --> Next Scene/Object position/Robot state
